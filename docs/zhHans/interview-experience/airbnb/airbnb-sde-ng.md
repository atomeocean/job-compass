## 第一轮：VO1（Coding）

### 题目：带截止时间的任务调度（贪心）

给定多组任务，每个任务有：

- id  
- deadline（天）  
- reward（积分）  

每个任务耗时一天。

**目标：**  
安排任务顺序，使所有被选任务在 deadline 前完成，并获得最大总 reward。  
输出执行顺序 + 最大奖励。

### 思路（贪心）

1. 按 reward 从高到低排序  
2. 对每个任务，从它的 deadline 往前找空位  
3. 最晚位置可放则放；找不到则跳过  
4. 最终提取所有被安排的任务

### 实现细节

- 用数组 `slots` 记录每天的安排  
- 长度为最大 deadline  
- 时间复杂度：O(N log N + N·D)  
- 空间复杂度：O(N + D)

### 面试中犯的错误

一开始把扫描方向写反 → 从前往后找导致错误。  
修正后 20 分钟完成并 run test 通过。


# 第二轮：VO2（Coding + Follow-up）

### Follow-up：如何优化？

当前最坏 O(N²)。  
可使用：

### **并查集（Union-Find）优化空槽查找**

- 每天建立一个母节点结构  
- find 用于快速跳到“前一个可用槽位”  
- union 在占位后更新链接  
- 复杂度接近线性 O(N α(N))


## 3. 解决方法与思考步骤

### 为什么贪心有效？

- 高 reward 优先  
- 放在最靠后的空位  
- 避免占用更早的天  
- 证明为经典最优策略

### 可扩展问题

- 任务不止一天怎么办？  
- deadline 极大怎么办？  
- reward 有负数怎么办？  

Airbnb 很喜欢继续追问。
