# Meta Software Engineer Phone Screen面试经验

<InterviewDetail />

## 基本信息

- **岗位**：Software Engineer
- **面试形式**：Phone Screen
- **申请渠道**：网上海投
- **候选人背景**：硕士，在职跳槽
- **面试结果**：未知

## 面试详情

整体过程比较顺利。

面试官全程口述题目要求，没有在共享文档里打字，因此刚开始需要花一些时间确认题意和边界条件。

理解题目之后发现难度并不算特别高，整体解题过程比较顺畅。

### 第一题

题目类似 LeetCode 924（Minimize Malware Spread）。

要求给出最优解法，并且空间复杂度需要达到：

```text
O(1)
```

面试过程中重点讨论了思路和复杂度优化。

### 第二题

题目类似 LeetCode 253（Meeting Rooms II）。

主要考察：

- 区间处理
- 排序
- 堆（Priority Queue）
- 时间复杂度分析

属于 Meta 比较常见的会议室类题目。

## 面试体验

两道题都属于 Medium 难度。

由于面试官全程口述题目，因此听题和确认需求会比直接阅读题目花费更多时间。

题目本身不算特别刁钻，重点还是：

- 快速理解需求
- 明确边界条件
- 写出正确解法
- 解释复杂度

整体发挥比较顺利。
