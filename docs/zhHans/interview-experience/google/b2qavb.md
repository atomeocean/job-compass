# Google L4 Software Engineer Onsite面试经验

<InterviewDetail />

## 基本信息

- **岗位**：Software Engineer（L4）
- **面试形式**：Onsite
- **申请渠道**：网上海投
- **候选人背景**：硕士，在职跳槽
- **面试结果**：Pass

## 面试详情

一共进行了 5 轮面试。

整体体验不错，不过整个流程推进比较慢。

### 第一轮：Coding

题目背景是 Google Calendar。

给定多个人的日程安排，找出所有人都没有时间冲突的可用时间段。

属于会议室（Meeting Rooms）系列题目的变体。

### 第二轮：Behavioral Interview

常规 BQ。

主要围绕过往项目经历、团队合作以及行为问题展开。

### 第三轮：Coding

题目与会议室系列比较类似，不过背景换成了租车系统。

模拟租车和还车流程，统计使用情况。

其中包括：

- 车辆使用记录
- 租车时间处理
- 找出使用次数最多的车辆

属于区间处理和统计类题目。

### 第四轮：Coding

题目属于经典 Topological Sort（拓扑排序）。

整体属于比较容易识别题型的一道图论题。

### 加面：AI / ML

额外安排了一轮 AI / ML 面试。

主要考察 LLM 相关基础知识，包括：

- Post Training
- Fine-tuning
- Prompt Engineering
- Transformer 基础原理

整体偏八股知识，不涉及现场实现模型或算法。

## 面试体验

整体难度不算高。

Coding 题大多属于经典题型或常见变体，重点还是：

- 快速识别题型
- 与面试官沟通思路
- 正确完成实现

AI / ML 加面主要考察基础知识储备。

整个流程体验不错，不过流程推进时间比较长。

## 面试结果反馈

- **最终结果**：Pass
