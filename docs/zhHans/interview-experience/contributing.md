# 面试经验贡献指南

欢迎为Job Compass贡献面试经验！您的分享将帮助更多求职者更好地准备面试。请遵循以下规则和流程。

## 贡献原则

1. **真实性**：请确保分享的面试经历是您亲身经历或经确认真实的。
2. **隐私保护**：
   - 请勿泄露面试官的真实姓名、联系方式或其他个人隐私信息。
   - 请勿泄露公司明确禁止分享的机密信息（如签署了NDA的特定数据）。
3. **客观友好**：描述面试过程时请保持客观，避免使用攻击性或侮辱性的语言。
4. **格式规范**：请尽量按照推荐的模板格式撰写，以便于阅读和归档。

## 贡献流程

### 1. 准备工作

如果您是首次贡献，请先Fork本仓库到您的GitHub账号，并Clone到本地。

### 2. 创建文件

在 `docs/zhHans/interview-experience` 目录下：

1. **选择或创建公司目录**：如果不确定公司目录名，请参考现有目录或创建一个新的英文目录名（全部小写，连字符分隔）。
2. **创建Markdown文件**：文件名建议使用 `YYMMDD-position.md` 或仅使用哈希/ID，尽量保持唯一性。

### 3. 撰写内容

请参考 [面经需求](./requirement.md) 中的字段建议。建议包含以下信息：

- **Metadata**（Frontmatter）：
  - 面试时间
  - 岗位类别
  - 公司名字
  - 面试结果

- **正文内容**：
  - 面试轮次及流程
  - 遇到的题目或考察点（如果是算法题，可简述题意）
  - 面试感受
  - 给后来者的建议

#### 示例模版

```markdown
---
title: Google SDE 实习面试
date: 2025-06-01
company: Google
position: SDE Intern
result: Offer
---

# Google SDE 实习面试分享

## 面试流程
- 第一轮：电面，聊项目...
- 第二轮：算法...

## 题目
1. 两数之和...

## 总结
面试体验很好，面试官很 nice...
```

### 4. 提交更改

1. 在本地 Commit 您的更改。
2. Push 到您的 GitHub 仓库。
3. 在 Job Compass 仓库页面发起 Pull Request (PR)。

我们会尽快 Review 您的 PR，合并后您的文章将出现在网站上。

感谢您的贡献！
