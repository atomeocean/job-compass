# Amazon AWS SDE面经分享

<InterviewDetail />

## 背景

这次面试的岗位是 AWS 相关方向。整体来说，面试中会涉及一些安全相关的问题，但深度不算特别高，更偏向工程实现、系统设计和 Amazon Leadership Principles 的考察。

## 面试时间线

* **第一轮**：OA，包含 AI Coding 部分。
* **第二轮**：Tech Screening，要求实现一个类似 NACLs 的 library。
* **Onsite**：两天内完成四轮面试，包括 System Design、两轮 Coding 和一轮 Behavioral Questions。

## 面试过程

### OA

第一轮 OA 的 AI Coding 部分会给一个代码库以及配套的 AI 工具。任务是根据 README 里的需求找到问题并修复代码。

一开始测试基本都是挂的，需要逐步修复问题，让测试全部通过。建议正式开始做题之前，先熟悉一下 sample，因为题目环境和流程需要一点适应时间。

我使用的是 Java。页面体验类似 VS Code，但不是很好用，所以提前熟悉环境会有帮助。

### Tech Screening

第二轮是 Tech Screening，题目是写一个类似 NACLs 的 library。

核心要求是根据不同的 rules 判断 request 应该 allow 还是 deny。整体思路类似网络访问控制列表，需要设计规则匹配和优先级处理逻辑。

### Onsite Loop

Onsite 分两天完成，一共四轮：

1. **Hiring Manager System Design**：设计一个管理上千台 device 的系统。
2. **Coding**：实现一个 Logger，按照时间顺序打印 message，可以使用 TreeMap。
3. **Coding**：实现一个 CLI，可以执行用户命令，并打印过去十个命令。
4. **Behavioral Questions**：纯 BQ 轮，主要围绕 Amazon Leadership Principles 展开，建议准备四个以上 LP。

## 题目整理

### System Design

System Design 轮由 Hiring Manager 面试，题目是设计一个可以管理上千台 device 的系统。

可以重点准备以下方向：

* Device 注册、鉴权和状态管理
* 批量配置下发
* 设备心跳和健康检查
* 日志与监控
* 故障恢复与告警
* 可扩展性和安全性

### Coding 1: Logger

题目是实现一个 Logger，要求按照时间顺序打印 message。

可以考虑使用 TreeMap 来维护时间戳和消息的映射关系。如果同一时间戳可能对应多个 message，需要处理重复时间戳的情况。

### Coding 2: CLI

题目是实现一个 CLI，可以执行用户输入的命令，并且支持打印过去十个命令。

这道题重点在于命令解析、历史记录维护和边界情况处理。可以使用队列或列表保存历史命令，并在超过十条时移除最早的记录。

### Behavioral Questions

BQ 是一整轮纯行为面试，基本是围绕 Amazon Leadership Principles 展开。面试过程更像干聊，但会深挖细节。

## 准备建议

* Amazon 一定要重点准备 Leadership Principles。
* 不要在多个 LP 中反复使用同一个例子。
* 至少准备十个不同的过往工作经历。
* 每个故事都要包含清晰的背景、目标、行动、结果和反思。
* 如果真实经历不够，也要保证故事细节完整、逻辑自洽。
* 技术部分要熟悉 Java 基础、数据结构、规则匹配、CLI 设计和常见系统设计模块。
* 安全相关内容不算特别深，但建议理解鉴权、访问控制、规则优先级和日志审计等基础概念。
* OA 前要熟悉 sample 和 coding 环境，避免因为页面不好用影响发挥。

## 总结

整体来说，Amazon AWS 面试既考察工程实现能力，也非常重视 Leadership Principles。

OA 重点是根据 README 修复代码并让测试通过。Tech Screening 关注规则匹配和访问控制。Onsite 则覆盖 System Design、Coding 和 BQ，其中 LP 依然是非常关键的部分。

建议大家在准备算法和系统设计的同时，提前打磨足够多样化的 LP 故事，并且注意每个故事的细节和真实性。祝大家都能顺利通过面试！
